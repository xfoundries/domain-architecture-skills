Rules:
- This is a package-role template, not a mandatory Maven module layout.
- domain contains aggregates, entities, value objects, domain events, domain services, aggregate repository contracts, and optional narrow secondary ports consumed by domain policies/services.
- application contains use cases, application services, primary ports, workflow/read secondary ports, commands, and query models.
- web contains primary adapters that call primary ports or application services. If inbound adapters are not mainly HTTP, rename this package to interface.
- infrastructure contains secondary adapters that implement secondary ports from domain or application modules. It contains no domain logic and no runtime framework configuration when a boot module exists.
- boot contains the runtime entry point, selected runtime framework starters, Spring configuration, component scanning, and bean wiring.
