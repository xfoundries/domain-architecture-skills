Rules:
- This is a package-role template, not a mandatory Maven module layout.
- domain contains aggregates, entities, value objects, domain events, domain services, aggregate repository contracts, and optional narrow secondary ports consumed by domain policies/services.
- application contains use cases, application services, primary ports, workflow/read secondary ports, commands, and query models.
- web contains primary adapters that call primary ports or application services. Keep HTTP/API Request/Response DTOs adapter-local under feature request/response packages. If inbound adapters are not mainly HTTP, rename this package to interface.
- infrastructure contains secondary adapters that implement secondary ports from domain or application modules. Group by technical shape first and business feature or external system second, such as persistence/aggregate-or-feature, query/feature, client/external-system, messaging/topic-or-system, file/feature, and cache/feature. Prefer query over readmodel unless the project explicitly uses read-model terminology. It contains no domain logic and no global runtime framework configuration when a runtime assembly package/module exists.
- boot contains the runtime entry point, selected runtime framework starters, runtime configuration, component scanning, and bean wiring. The package name may differ outside Spring Boot projects.
